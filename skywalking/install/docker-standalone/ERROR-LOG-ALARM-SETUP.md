# ERROR Log Alarm Configuration for fuzzy-train-java

This document explains how ERROR log detection and alerting is configured for the fuzzy-train-java service.

## Overview

The setup uses three components to detect ERROR logs and send alerts to Microsoft Teams:

1. **LAL (Log Analysis Language)** - Parses logs and extracts log levels
2. **MAL (Meter Analysis Language)** - Aggregates log metrics
3. **Alarm Rules** - Triggers alerts based on metric thresholds

## Architecture

```
Fuzzy-Train Java
      ↓
   (sends logs)
      ↓
  OAP Server
      ↓
   LAL Parser ────→ Extracts "ERROR" from log text
      ↓
  Generates Metric: log_count_by_level{level='ERROR'}
      ↓
   MAL Engine ────→ Aggregates to: log_error_count
      ↓
  Alarm Engine ───→ Evaluates rules
      ↓
   (if threshold exceeded)
      ↓
Microsoft Teams Webhook
```

## Configuration Files

### 1. LAL Configuration (`config/lal-config.yaml`)

Parses incoming logs and extracts the log level:

```yaml
rules:
  - name: fuzzy-train-log-analysis
    layer: GENERAL
    dsl: |
      filter {
        json {
          abortOnFailure false
        }

        extractor {
          service log.service
          layer 'GENERAL'

          // Detect ERROR in log body text
          if (log.body.text.matches(".*ERROR.*")) {
            tag level: "ERROR"
            metrics {
              name "log_count_by_level"
              timestamp log.timestamp
              labels level: "ERROR", service: log.service
              value 1
            }
          }
        }

        sink {}
      }
```

**How it works:**
- Receives logs from fuzzy-train-java
- Checks if log body contains "ERROR" text
- Creates a metric `log_count_by_level` with label `level=ERROR`
- Each ERROR log increments the counter by 1

### 2. MAL Configuration (`config/log-mal-rules.yaml`)

Aggregates the raw metrics into usable alarm metrics:

```yaml
metricsRules:
  - name: error_count
    exp: log_count_by_level.tagEqual('level', 'ERROR').sum(['service', 'instance']).increase('PT1M')
```

**How it works:**
- Filters metrics where `level='ERROR'`
- Sums by service and instance
- Calculates increase over 1 minute window
- Result: `log_error_count` metric

### 3. Alarm Rules (`config/alarm-settings.yml`)

Defines when to trigger alerts:

```yaml
rules:
  # Immediate alert on ANY ERROR
  fuzzy_train_java_error_detected_rule:
    expression: log_error_count{service='fuzzy-train-java'} > 0
    period: 1
    silence-period: 3
    message: "🚨 ERROR detected in fuzzy-train-java service!"
    tags:
      level: CRITICAL

  # Alert on high ERROR count
  fuzzy_train_java_high_error_count_rule:
    expression: log_error_count{service='fuzzy-train-java'} > 5
    period: 1
    silence-period: 5
    message: "⚠️ High ERROR count: More than 5 ERROR logs in last minute."
    tags:
      level: CRITICAL

hooks:
  webhook:
    teams-alerts:
      is-default: true
      urls:
        - https://default3d08c29d04bc44cca8673238f9d6e4.01.environment.api.powerplatform.com:443/...
```

**How it works:**
- **Rule 1**: Triggers if ANY ERROR log appears (> 0)
  - Checks every 1 minute (`period: 1`)
  - Won't trigger again for 3 minutes (`silence-period: 3`)

- **Rule 2**: Triggers if more than 5 ERROR logs in 1 minute
  - Checks every 1 minute
  - Won't trigger again for 5 minutes

- **Webhook**: Sends alert to Microsoft Teams

## Docker Compose Configuration

The `docker-compose.fuzzy-train.yml` mounts these configs:

```yaml
oap:
  environment:
    SW_LOG_LAL_FILES: lal-config
    SW_LOG_LAL_DIR: /skywalking/config/lal
    SW_LOG_MAL_FILES: log-mal-rules
  volumes:
    - ./config/lal-config.yaml:/skywalking/config/lal/lal-config.yaml:ro
    - ./config/log-mal-rules.yaml:/skywalking/config/log-mal-rules/log-mal-rules.yaml:ro
    - ./config/alarm-settings.yml:/skywalking/config/alarm-settings.yml:ro
```

## Testing the Setup

### 1. Verify OAP is healthy

```bash
curl http://localhost:12800/healthcheck
# Should return: 200 OK
```

### 2. Check if services are registered

```bash
curl -s -X POST http://localhost:12800/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ listServices(layer: \"GENERAL\") { name } }"}' | jq
```

### 3. View fuzzy-train-java logs

```bash
docker compose -f docker-compose.fuzzy-train.yml logs fuzzy-train-java --tail=20
```

You should see logs with ERROR level being generated.

### 4. Wait for metrics to be generated

LAL and MAL process logs in batches. Wait 1-2 minutes for metrics to appear.

### 5. Check if metrics are being generated

```bash
# Query for log metrics (this endpoint may not be available in all versions)
curl -s http://localhost:12800/internal/metrics
```

### 6. Monitor for Teams alerts

- ERROR logs are being generated by fuzzy-train-java continuously
- Within 1-2 minutes, you should receive a Teams notification
- Check your Microsoft Teams channel for the alert

## Troubleshooting

### No alerts received

1. **Check OAP logs for LAL/MAL errors:**
   ```bash
   docker compose -f docker-compose.fuzzy-train.yml logs oap | grep -i "lal\|mal\|error"
   ```

2. **Verify LAL config is loaded:**
   ```bash
   docker compose -f docker-compose.fuzzy-train.yml exec oap ls -la /skywalking/config/lal/
   ```

3. **Verify MAL config is loaded:**
   ```bash
   docker compose -f docker-compose.fuzzy-train.yml exec oap ls -la /skywalking/config/log-mal-rules/
   ```

4. **Check alarm config:**
   ```bash
   docker compose -f docker-compose.fuzzy-train.yml exec oap cat /skywalking/config/alarm-settings.yml | grep -A 10 "fuzzy_train_java"
   ```

5. **Test Teams webhook manually:**
   ```bash
   curl -X POST \
     'https://default3d08c29d04bc44cca8673238f9d6e4.01.environment.api.powerplatform.com:443/powerautomate/automations/direct/workflows/eedba587755f4a1ab3bfc24043ecca0d/triggers/manual/paths/invoke?api-version=1&sp=%2Ftriggers%2Fmanual%2Frun&sv=1.0&sig=KQe6e02uzatx84L94u-iUDjyg56UQm7pxveo91giPtY' \
     -H 'Content-Type: application/json' \
     -d '[{"scopeId":1,"scope":"SERVICE","name":"fuzzy-train-java","id0":"test","id1":"","ruleName":"test_rule","alarmMessage":"Test alert from Docker","startTime":1234567890000,"tags":[{"key":"level","value":"CRITICAL"}]}]'
   ```

### Metrics not appearing

1. **Check if logs are being received:**
   - Open SkyWalking UI: http://localhost:8080
   - Go to "General Service" layer
   - Click on "fuzzy-train-java"
   - Go to "Log" tab
   - You should see logs appearing

2. **Verify log format:**
   - LAL expects logs to contain "ERROR" text
   - Check fuzzy-train-java logs to confirm ERROR logs are being generated

3. **Restart OAP if needed:**
   ```bash
   docker compose -f docker-compose.fuzzy-train.yml restart oap
   ```

## Customization

### Change ERROR threshold

Edit `config/alarm-settings.yml`:

```yaml
fuzzy_train_java_error_detected_rule:
  expression: log_error_count{service='fuzzy-train-java'} > 10  # Change from 0 to 10
```

### Change time window

Edit `config/log-mal-rules.yaml`:

```yaml
- name: error_count
  exp: log_count_by_level.tagEqual('level', 'ERROR').sum(['service', 'instance']).increase('PT5M')  # Change from PT1M to PT5M (5 minutes)
```

### Add more services

The LAL rule applies to all services in the GENERAL layer. To monitor another service:

```yaml
# In alarm-settings.yml
another_service_error_rule:
  expression: log_error_count{service='another-service'} > 0
  period: 1
  silence-period: 5
  message: "ERROR detected in another-service!"
```

### Change alert message

Edit `config/alarm-settings.yml`:

```yaml
message: "🔥 CRITICAL: {name} has ERROR logs! Investigate immediately!"
```
